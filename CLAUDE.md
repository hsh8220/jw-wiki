# JW Wiki — 스키마 및 운영 규칙

여호와의 증인 온라인 출판물(**jw.org**, **wol.jw.org**)의 내용을 수집하고 지식화하는
개인 위키다. [llm-wiki.md](llm-wiki.md)의 패턴을 이 도메인에 맞게 구체화한 것이다.

이 문서는 LLM 에이전트(너)가 이 위키를 **규율 있게 유지보수**하도록 만드는 설정 파일이다.
새 세션을 시작하면 이 파일과 [index.md](index.md), [log.md](log.md)를 먼저 읽어라.

---

## 1. 3계층 구조

| 계층 | 위치 | 소유자 | 설명 |
|------|------|--------|------|
| **원본(raw)** | `raw/` | 도구 | 사이트에서 수집한 불변 원본. 절대 수정하지 않는다. |
| **위키(wiki)** | `wiki/` | LLM | 요약·개념·인물·성구 페이지. LLM이 전적으로 작성/유지. |
| **스키마** | `CLAUDE.md` | LLM+사용자 | 이 문서. 규칙과 워크플로. 함께 진화시킨다. |

```
jw_wiki/
├── CLAUDE.md              # 이 파일 (스키마)
├── index.md              # 위키 전체 카탈로그 (내용 중심)
├── log.md                # 시간순 작업 기록 (append-only)
├── tools/
│   ├── jwfetch.py        # URL → raw 수집 파이프라인 CLI (동영상·기사·카테고리)
│   ├── jwlib.py          # 수집 공용 라이브러리 (API·추출·자막 폴백)
│   ├── jwscripture.py    # 전사본에서 성구 추출 + 한국어 정경 정규화(영어→한국어)
│   ├── gen_talk_pages.py # raw 동영상 → 소스·연사·성경책 페이지·index 일괄 생성
│   └── wikisearch.py     # 위키 전문 검색(grep 기반)
├── raw/
│   ├── articles/         # 기사 원본 (.md, frontmatter + 본문)
│   ├── videos/           # 동영상 원본 (.md, frontmatter + 자막 전사본)
│   └── assets/           # 내려받은 미디어(선택)
└── wiki/
    ├── overview.md       # 위키 진입점 / 큰 그림
    ├── sources/          # 원본 1건당 요약 페이지 1개
    ├── concepts/         # 교리·주제 개념 페이지 (평화, 하느님의 왕국 …)
    ├── entities/         # 인물·장소·조직 (예수, 사도 바울, 길르앗 학교 …)
    └── scriptures/       # 성구 페이지 (요한복음 14:27 …)
```

## 2. 수집(Ingest) 워크플로 — URL을 받으면

사용자가 jw.org/wol.jw.org URL을 주면 다음을 수행한다.

1. **수집 도구 실행** — raw 파일을 생성한다.
   ```bash
   .venv/bin/python tools/jwfetch.py "<URL>"          # 동영상·기사·카테고리 자동 판별
   .venv/bin/python tools/jwfetch.py "<카테고리 URL>"  # 그 안의 모든 동영상 일괄 수집
   ```
   - 동영상 URL → `raw/videos/<slug>.md` (메타데이터 + 자막 전사본)
   - 기사 URL → `raw/articles/<slug>.md` (메타데이터 + 본문 마크다운 + 성구 목록)
   - **카테고리 URL**(`#.../categories/<KEY>`) → 그 카테고리의 모든 동영상 수집.
     이미 받은 항목은 건너뛴다(`--force`로 강제, `--limit N`으로 개수 제한).
   - **자막 폴백**: 한국어 자막(.vtt)이 없으면 자동으로 영어 자막을 받아 전사한다.
     이 경우 frontmatter `transcript_lang: E`로 표시된다(전 언어 미발행이면 빈 전사본).
   - 여러 URL을 한 번에 줄 수 있다. `--json`으로 결과 메타데이터만 받을 수도 있다.

2. **원본 읽기** — 생성된 raw 파일을 읽고 핵심 내용을 파악한다.
   동영상이면 자막 전사본을, 기사면 본문을 정독한다.

3. **사용자와 핵심 요약 공유** — 무엇을 강조할지 짧게 합의한다(대화형 수집).
   배치 수집 시에는 생략 가능.

4. **소스 페이지 작성** — `wiki/sources/<slug>.md` 생성 (§4 형식).
   - **대량(카테고리) 수집**: `tools/gen_talk_pages.py`로 소스 페이지·연사 페이지·
     성경책 페이지·index를 한 번에 일괄 생성한다. 이때 요약은 제목의 '주제' + 전사본
     도입부(추출형)다. 특정 연설의 **심층 요약**이 필요하면 그 전사본을 직접 읽어
     소스 페이지의 `## 핵심 내용`을 손으로 채운다.
   - **소량(1~2건) 수집**: 전사본/본문을 정독해 처음부터 손으로 요약 페이지를 쓴다.

5. **위키 전반 갱신** — 이 소스가 다루는:
   - 개념 페이지(`wiki/concepts/`)를 갱신하거나 신규 생성
   - 인물/장소 페이지(`wiki/entities/`) 갱신/생성
   - 성구 페이지(`wiki/scriptures/`) 갱신/생성 — JW 도메인 특성상 **성구가 핵심 연결고리**다
   - 기존 페이지와 모순되면 명시적으로 표기한다(§6).
   한 소스가 보통 위키 페이지 5~15개를 건드린다.

6. **index.md, log.md 갱신** (§5).

7. **raw frontmatter의 `ingested: false` → `true`** 로 바꾼다 (이 한 줄만 수정 허용).

> 핵심: **도구는 깨끗한 원본을 만들 뿐, 지식화(요약·교차참조·정리)는 LLM의 일이다.**

## 3. 질의(Query)와 점검(Lint)

- **질의**: 질문을 받으면 `index.md`를 먼저 읽어 관련 페이지를 찾고, 드릴다운해
  인용과 함께 답한다. 가치 있는 답(비교표·분석·새 연결)은 위키 페이지로 **되돌려 저장**한다.
- **점검**: 요청 시 `tools/wikisearch.py`와 index를 활용해
  모순·낡은 주장·고아 페이지·교차참조 누락·개념 페이지가 없는 핵심 용어를 찾아 보고한다.

## 4. 페이지 형식 (frontmatter 규약)

모든 위키 페이지는 YAML frontmatter로 시작한다. (Obsidian Dataview 호환)

**소스 요약** `wiki/sources/<slug>.md`:
```markdown
---
type: source
source_type: video | article
title: "원제목"
source_url: <원본 URL>
raw: ../../raw/videos/<slug>.md   # 원본 파일 상대경로
date: 2026-06-06
scriptures: ["요한복음 14:27", ...]   # 기사인 경우
tags: [평화, 예수, ...]
---

# 제목

## 한 줄 요약
## 핵심 내용  (불릿)
## 주요 성구  ([[scriptures/요한복음 14_27]] 식 링크)
## 관련 페이지 ([[...]] 위키링크)
## 출처  (원본 링크 + raw 경로)
```

**개념/인물/성구 페이지**: `type: concept | entity | scripture`, `aliases`,
`tags`, 그리고 본문에 정의/설명 + `## 등장 소스`(이 항목을 다룬 소스 [[링크]] 목록).

**연결 규칙**:
- 페이지 간 연결은 Obsidian 위키링크 `[[경로/페이지명]]`을 쓴다. 링크는 **후하게** 건다.
- **연사 페이지**: `wiki/entities/<연사명>.md` (공백 보존, 예: `M. 스티븐 레트.md`).
  한 연사의 모든 연설을 연대순으로 모은다. `gen_talk_pages.py`가 자동 생성/갱신.
- **성구 페이지 — 두 종류 공존**:
  - *책 단위* `wiki/scriptures/<책>.md` (예: `시편.md`) — 대량 수집의 기본.
    그 책에서 인용된 구절들과 인용한 소스를 집계. `gen_talk_pages.py`가 생성.
  - *구절 단위* `wiki/scriptures/<책 장_절>.md` (예: `요한복음 14_27.md`) — 특정 구절을
    심층 해설할 때 손으로 만든다(파일명의 `:`→`_`, 본문 표기는 `요한복음 14:27`).
- 소스/원본 파일 슬러그는 한글을 보존한다(도구의 `slugify`와 동일 규칙).
- `index.md`는 `gen_talk_pages.py`가 `wiki/`를 스캔해 자동 재생성한다(직접 편집 금지).

## 5. 인덱스와 로그

- **index.md** — 내용 카탈로그. 카테고리별로 페이지 1줄씩(링크 + 한 줄 요약).
  매 수집마다 갱신.
- **log.md** — 시간순 기록. 항목은 반드시 다음 접두사로 시작한다(grep 가능):
  ```
  ## [YYYY-MM-DD] ingest | <제목>
  ## [YYYY-MM-DD] query  | <질문 요지>
  ## [YYYY-MM-DD] lint   | <점검 요지>
  ```
  `grep "^## \[" log.md | tail -5` 로 최근 활동 확인.

## 6. 도메인 규칙 (여호와의 증인 출판물)

- **중립적·사실 기반 서술**: 출판물의 가르침을 *그 출판물이 말하는 대로* 정확히 요약한다.
  위키는 출판물 내용의 지식 베이스이지 논평이 아니다. 평가/반박을 덧붙이지 않는다.
- **용어**: 출판물의 표기를 따른다 — "여호와", "하느님의 왕국", "신세계역" 등.
- **성구 중심**: 거의 모든 기사가 성구로 논지를 전개한다. 성구 페이지는
  "어떤 소스에서 어떤 맥락으로 인용됐는가"를 누적한다 — 이것이 이 위키의 가장 강력한 그래프다.
- **번역본 구분**: 같은 구절의 「신세계역」과 다른 번역(새번역 등) 인용을 구분해 기록한다.
- **동영상**: 자막 전사본이 본문 역할을 한다. 화자/장면 전환이 불명확할 수 있으니
  요약 시 전사본 근거를 분명히 한다(시점·인용 표시).
- **모순/갱신**: 새 소스가 기존 페이지와 다르면 삭제하지 말고
  `> ⚠️ <날짜> <소스>에 따르면 …` 형태로 병기하고 출처를 남긴다.

## 7. 환경

- 의존성: `requirements.txt`. 가상환경은 `.venv/` (이미 생성됨).
  설치: `.venv/bin/pip install -r requirements.txt`
- 도구 실행은 항상 `.venv/bin/python`을 쓴다(시스템 파이썬엔 의존성이 없다).
- 위키는 git 저장소다 — 의미 있는 단위로 커밋하면 변경 이력을 얻는다.
